제일 중요한건 내가 요구하는 기능을 구성할때는 msa철학에 따라 
1서버 1db를 구성해서 최대한 분리성이 뛰어나게 구성해줘 차후에
확장이 편할 수 있게

1. 현재 구성된  msa 시스템을 nginx > gateway > 각 서버 
로 구성되는데 msa 정석대로 각 역할대로 분리해서 구성해주고
인증 인가를 auth-service에서 할 지 gateway에서 일괄로 하는 지에
대해서는 정확히 정석대로 해주고, oauth 기능도 추가할거야 
이 과정에서 현재 제대로 값이 .env로 들어가지 않고 하드 코딩으로
된 부분들을 점검해서 보안이 뛰어나게 다시 구성도 해줘
값은 내가 운영서버에서 쓰던 값 그대로 써줘 
나중에 알아서 쓰라고 하지말고

2. 검색 기능도 추가할건데 search-service를 추가해서 의미 검색
통합 기능을 구축할거야 vector 임베딩 > vector search랑 의미 
기반 검색 시스템을 구축할거야 무료 임베딩 모델이랑 무료 모델을 활용해서 
서치할거야 이건 완전 무료인 모델을 바탕으로 구성해줘 

3. 현재 개발,운영 서버 모두 각각 띄우는데 개발 서버를 수정하고
테스트 후에 ci/cd 자동화 테스트 후 바로 운영서버로 올려서 
운영하는 방식으로 재구성하고 싶어  

4.이제 서버가 많아져서 로그를 다 각각 확인하기 어려운데
elk를 도입해서 전체적으로 도입해서 모든 로그들을 통합 관리하는
기능도 추가해줘

5.백오피스를 만들어서 전체를 관리하는 페이지도 구성해줘

6. 각 구성을 자동으로 테스트하는 부분도 구성해줘

7.각 구성하는 기능에 대한 페이지를 구성한 후 UI/UX를 그에 맞게
최대한 깔끔하게 구성해줘 기능에 문제 없게 해주고 그리고 seo가
최적화 될 수 있게 구성해주고 

8.1~7를 구성 후에 각 기능에 대한 설명에 대한 문서 작성도
부탁하고 구성 도중에 내가 제시한 요구보다 더 좋은 기능이 있다면
그거에 맞게 추가해서 구성해줘 그리고 구성을 마친 후에는 
전체 기능을 제대로 한 번 다 점검해주고 문제점을 수정해줘
